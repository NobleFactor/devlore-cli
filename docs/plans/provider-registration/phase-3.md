# Phase 3: Codegen Templates

## Summary

Update the `star` code generation templates to emit a single provider
descriptor per provider instead of three separate `init()` functions calling
`RegisterBinding`. After regeneration, each provider package has one
`init()` that calls `op.Announce()`.

## Current State (3 init functions per provider)

Each generated provider currently has three files, each with its own
`init()` calling `op.RegisterBinding` with a partial `ProviderBinding`:

| File | init() registers |
| --- | --- |
| `actions.gen.go` | `ActionRegistrar` function pointer |
| `planned.gen.go` | `PlannedFactory` function pointer + `Access`/`Lifetime` |
| `immediate.gen.go` | `ImmediateFactory` function pointer + `Access`/`Lifetime` |

The `RegisterBinding` merges these partials by provider name at import time.

## Target State (1 descriptor per provider)

A single generated file (`provider.gen.go`) contains the provider
descriptor struct implementing `op.Provider` (and optionally
`op.PlannedProvider`, `op.ImmediateProvider`):

```go
// Code generated by go.render; DO NOT EDIT.

package file

import (
    "reflect"

    "go.starlark.net/starlark"

    "github.com/NobleFactor/devlore-cli/pkg/op"
    provider "github.com/NobleFactor/devlore-cli/pkg/op/provider/file"
)

type fileProvider struct{}

func (p *fileProvider) Name() string { return "file" }

func (p *fileProvider) Register(reg *op.ActionRegistry, ctx op.Context) {
    impl := &provider.Provider{}
    op.InitProvider(impl, ctx)
    op.RegisterReflectedActions(reg, "file", impl, Params)
}

func (p *fileProvider) NewPlanned(graph *op.Graph, project string, reg *op.ActionRegistry) starlark.Value {
    return NewFilePlan(graph, project, reg)
}

func (p *fileProvider) NewImmediate(cfg op.BindingConfig) starlark.Value {
    // ... provider-specific construction
}

func init() {
    op.Announce(&fileProvider{})
}
```

The existing `NewFilePlan` and `NewFileReceiver` helper functions stay ‚Äî
they are called by the descriptor methods.

## Template Changes

### 1. New template: `provider_descriptor.go.template`

Emits the provider descriptor struct with:
- `Name()` returning the provider name
- `Register()` containing the current `ActionRegistrar` body
- `NewPlanned()` (if provider has planned access) containing the current `PlannedFactory` body
- `NewImmediate()` (if provider has immediate access) containing the current `ImmediateFactory` body
- `init()` calling only `op.Announce()`

### 2. Remove init() from existing templates

- `graph_actions.go.template` ‚Äî remove the `init()` / `RegisterBinding` block; keep `Params`
- `planned_receiver.go.template` ‚Äî remove the `init()` / `RegisterBinding` block; keep `NewXxxPlan`
- `immediate_receiver.go.template` ‚Äî remove the `init()` / `RegisterBinding` block; keep `NewXxxReceiver`

### 3. Regenerate all providers

Run `star devlore actions generate` to regenerate all 42 provider files.
Verify each generated package has exactly one `init()` with exactly one
`op.Announce()` call.

## Exit Criteria

- All generated providers use `op.Announce()` ‚Äî zero calls to `op.RegisterBinding`
- Each generated package has exactly one `init()` function
- Generated provider descriptors implement `Provider` (and optionally `PlannedProvider`/`ImmediateProvider`)
- Helper functions (`NewXxxPlan`, `NewXxxReceiver`, `Params`) still exist and are called by the descriptor
- All tests pass
