## Context

Phases 2Aâ€“2D restructured the execution layer: Actions have `Do(ctx, slots)`/
`Undo(ctx, slots, state)` signatures, actions live in per-provider packages
(`provider/file`, `provider/git`, etc.), nodes carry Action interface instances
(not strings), planned receivers thread `ActionRegistry` and use `reg.MustGet()`.

The `_gen.go` files are currently **hand-written** to match this new contract.
The code generation templates (`graph_ops.go.tmpl`, `planned_receiver.go.tmpl`)
and the helper functions in noblefactor-ops (`receiver_go_gen.go`) still emit
the **old** pattern: `Execute(ctx, node) error`, `node.GetSlot(...)`,
`Operation: "name"`, `[]Operation` factory.

Phase 3 updates both repos so the templates produce output matching the
hand-written files. After this phase, `rm *_gen.go` + re-run
`star gen.receiver` reproduces identical output.

**Repos**: noblefactor-ops (template engine + helpers), devlore-cli (template files)

## Current vs Target

### graph_ops (actions_gen.go)

| Aspect | Current Template | Target |
|---|---|---|
| Package | `execution` | Per-provider (e.g., `file`) |
| Struct name | `{{Struct}}{{Method}}Op` (e.g., `FileBackupOp`) | `{{Method}}` (e.g., `Backup`) |
| Impl field | `impl *{{ImplType}}` (unexported) | `Impl *Provider` (exported) |
| Execute | `Execute(ctx *Context, node *Node) error` | `Do(ctx *execution.Context, slots map[string]any) (execution.Result, execution.UndoState, error)` |
| Undo | (absent) | `Undo(ctx *execution.Context, slots map[string]any, state execution.UndoState) error` |
| Slot reads | `node.GetSlot("source").(string)` | `slots["source"].(string)` |
| Content | `ctx.ContentFor(node)` / `ctx.StoreContent(node, data)` | `slots["content"].([]byte)` / return as Result |
| Dry-run return | `return nil` | `return nil, nil, nil` |
| Delegation return | `return o.impl.Method(args...)` | `return nil, nil, o.Impl.Method(args...)` |
| Factory | `{{Struct}}Ops(impl) []Operation` | `Register(reg *execution.ActionRegistry)` |
| Import | `"fmt"` | `"fmt"` + `"github.com/.../execution"` |

### planned_receiver (planned_*_gen.go)

| Aspect | Current Template | Target |
|---|---|---|
| Struct | `Receiver` embed, graph, host, project | Same + `reg *execution.ActionRegistry` |
| Constructor | `New{{Struct}}Plan(graph, h, project)` | `New{{Struct}}Plan(graph, h, project, reg)` |
| Node.Action | `Operation: "category.method"` (string) | `Action: p.reg.MustGet("category.method")` |
| Attr helper | `MakeAttr(...)` | Same |
| FillSlot | `FillSlot(node, p.graph, name, val)` | Same |

## Design

### noblefactor-ops: Template Helper Changes

**File**: `internal/starlark/receiver_go_gen.go`

The template helpers are registered in `genTemplateFuncs`. Changes needed:

#### 1. `tplGraphReaders` â€” slot reads from `slots` map

Current: `source := node.GetSlot("source").(string)`
Target: `source := slots["source"].(string)` (with `, _ :=` for optional types)

- Remove content pipeline handling (`ContentFor`/`StoreContent`)
- Context-provided params (`io.Writer`) read from `ctx.Logger` (unchanged)
- Engine-injected params (func types) read from `slots[...]` (unchanged)

#### 2. `tplGraphReturn` â€” three-value return

Current: `return o.impl.Method(args...)` or error handling
Target: `return nil, nil, o.Impl.Method(args...)` (for error-only returns)

For content producers (return `[]byte`):
- Current: `StoreContent(ctx, node, result)`; return nil
- Target: `return result, nil, nil`

For content consumers (accept `[]byte`):
- Already handled by slot reads (content is a slot, not ContentFor)

#### 3. `tplDryRunChecksum` â€” update for new return

Current: `ctx.TargetChecksum = ChecksumBytes(content)`; (return nil already)
Target: Same checksum line, but dry-run block ends with `return nil, nil, nil`

#### 4. `tplImplArgs` â€” capitalized `Impl`

Change `o.impl.` â†’ `o.Impl.` in delegation calls.

#### 5. New helper: `tplGraphUndo` â€” Undo method stub

Generate:
```go
func (o *{{Name}}) Undo(_ *execution.Context, _ map[string]any, _ execution.UndoState) error {
    return nil
}
```

#### 6. `tplPlanFillSlots` â€” unchanged (already correct)

#### 7. `tplPlanUnpackArgs` â€” unchanged (already correct)
