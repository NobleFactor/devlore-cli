## Outstanding / open questions

- RetryPolicy `nil` semantics flip (today `nil`="no retry"; proposed `nil`="inherit") — a behavioral change to confirm.
- The RetryPolicy floor value (interacts with the `nil`-flip).
- Elevation's split across plan time vs execution time, given the requirement is host-resolved (`platform.Detect`).
- Whether the elevated **session** becomes a first-class runtime object; pause-on-approval reuse of `ErrPaused`.

## Discrepancies (design vs. current code)

- **RetryPolicy** is the only fully-wired member; consumed solely at `ActivationRecord.DispatchChild`. The **root
  subgraph's own policy is inert** (root dispatched directly by `GraphExecutor.Run`, never via `DispatchChild`).
- **ErrorAction** is consumed as a best-effort **observation hook** (result/error discarded, original error always
  surfaces) — *not* a recovery path, and distinct from compensation. Its accessor doc claims a `flow.failed` sentinel
  default, but that default is **deferred/unwired** (`nil` currently = skip).
- **ElevationOffer** is **plumbed but dormant**: never read, not serialized, no starlark path; `flow.Provider.Elevate()`
  is an empty no-op. The only live elevation is the per-manager `NeedsSudo()` boolean → inline `sudo -n`, explicitly
  TODO'd to route through `ElevationOffer`.
- Inheritance for retry is **not implemented** — the per-child lookup is flagged "interim" with a pending frame-chain
  helper; this design replaces that with plan-time resolution.
