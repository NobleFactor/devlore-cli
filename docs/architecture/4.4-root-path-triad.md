# Root, Path, and RecoverySite — The Cooperative Triad

`op.Root`, `op.Path`, and `op.RecoverySite` form a cooperative triad that
governs all filesystem I/O in the system. Root produces and consumes Path.
RecoverySite accepts Path from providers and delegates to Root. Path is the
universal currency flowing between all participants.

## 2. Roles

```
  ┌─────────┐         ┌─────────┐         ┌──────────────┐
  │  Root   │produces │  Path   │flows to │ RecoverySite │
  │         │────────▶│         │────────▶│              │
  │ factory │         │currency │         │ orchestrator │
  │ engine  │◀────────│         │         │              │
  │         │consumed │         │         │              │──┐
  └─────────┘   by   └─────────┘         └──────────────┘  │
       ▲                                        │           │
       │              delegates to               │           │
       └─────────────────────────────────────────┘           │
                                                             │
            produces Paths internally for recovery ──────────┘
```

**Root** — Factory and engine. Produces Path via `NewPath`. Consumes Path
for all I/O operations. The interface has three implementations that
determine the access mode; provider code never knows which one is active.

**Path** — Universal currency. An immutable value type carrying
`{root, rel, abs}` where `abs` is derived as `filepath.Join(root, rel)`.
Flows between all participants. Serializes as `{root, rel}` — abs is
derived on deserialization. Created only through `Root.NewPath` or
`op.NewPath` (tests/deserialization).

**RecoverySite** — Orchestrator. Accepts Path from providers for
user-facing file locations. Returns opaque string recovery IDs (not Paths)
for tombstone storage. Delegates all I/O to `ctx.Root` internally,
producing its own Paths for the `.devlore/recovery/` directory.

## 3. Data Flow: Write with Recovery

```
  Provider                 RecoverySite              Root                Path
     │                         │                      │                   │
     │  root.NewPath("f.txt")  │                      │                   │
     │────────────────────────────────────────────────▶│                   │
     │                         │                      │──── produce ─────▶│
     │◀───────────────────────────────────────────────────────────────────│
     │  p = Path{root, "f.txt", abs}                  │                   │
     │                         │                      │                   │
     │  site.ArchiveFile(p)    │                      │                   │
     │────────────────────────▶│                      │                   │
     │                         │  root.NewPath(        │                   │
     │                         │   ".devlore/recovery/ │                   │
     │                         │    <uuid>")           │                   │
     │                         │─────────────────────▶│                   │
     │                         │                      │──── produce ─────▶│
     │                         │◀─────────────────────────────────────────│
     │                         │  recoveryPath = Path  │                   │
     │                         │                      │                   │
     │                         │  root.MkdirAll(dir)  │                   │
     │                         │─────────────────────▶│                   │
     │                         │                      │◀── p.Rel() ──────│
     │                         │                      │    or p.Abs()     │
     │                         │                      │  (mode-dependent) │
     │                         │                      │                   │
     │                         │  root.Rename(p,       │                   │
     │                         │    recoveryPath)     │                   │
     │                         │─────────────────────▶│                   │
     │                         │                      │◀── both Paths ───│
     │                         │                      │                   │
     │  recoveryID = "<uuid>"  │                      │                   │
     │◀────────────────────────│                      │                   │
     │                         │                      │                   │
     │  root.WriteFile(p, data)│                      │                   │
     │────────────────────────────────────────────────▶│                   │
     │                         │                      │◀── p.Rel() ──────│
     │                         │                      │    or p.Abs()     │
     │                         │                      │                   │
     ▼                         ▼                      ▼                   ▼
```

The provider never constructs a recovery Path. It passes its `op.Path` to
RecoverySite, which handles the `.devlore/recovery/<uuid>` layout internally.
The provider receives an opaque string ID and stores it in the tombstone.
During compensation, the provider passes the original Path and the recovery
ID back — RecoverySite reconstructs the recovery Path and renames the file
back.

## 4. Mode Switch

The mode switch is invisible to providers and RecoverySite. Swap the Root
implementation at startup and the entire system changes behavior:

| Implementation | Path dispatch | Confinement | Use case |
| --- | --- | --- | --- |
| `confinedRoot` | `p.Rel()` → `*os.Root` | Kernel-enforced | Execution |
| `RootReader` | `p.Abs()` → `os.*` | None (reads only) | Planning |
| `RootReaderWriter` | `p.Abs()` → `os.*` | None | Testing |

The composition chain — `rootBase` → `RootReader` → `RootReaderWriter` —
means `RootReaderWriter` inherits all read methods from `RootReader` and
overrides only the write stubs. `RootReader` returns `ErrReadOnly` for
all write operations, enforcing the read-only planning contract at the
application layer.

`confinedRoot` wraps `*os.Root` (Go 1.24). The kernel blocks symlink
escapes and `..` traversal. Paths outside the authority boundary are
rejected. This is the only implementation used in production execution.

## 5. Path as Serialization Boundary

Path serializes as `{root, rel}`. Abs is derived on deserialization:

```json
{"root": "/project", "rel": "src/main.go"}
```

This is the canonical form for persistence (recovery stacks, tombstones,
resource catalogs). The `root` field matches `os.Root.Name()` — it records
the authority boundary at the time the Path was created. On a different
machine with a different base directory, a new Root produces Paths with a
different `root` value. The graph is portable; the Paths are not.

Recovery IDs are opaque strings (UUIDs), not Paths. The RecoverySite
owns the `.devlore/recovery/` layout. Providers store the ID; only
RecoverySite knows how to turn it back into a Path.

## 6. Design Properties

1. **Mode-agnostic providers.** Provider code calls `root.ReadFile(p)`,
   `root.Stat(p)`, `site.ArchiveFile(p)`. It never branches on the Root
   implementation. The caller (executor, planner, test harness) picks the
   implementation at startup.

2. **Path is immutable.** Once created, a Path's fields never change.
   It is safe to store, serialize, pass across goroutines, and embed in
   resources and tombstones.

3. **Root is the single factory.** Paths are created only through
   `Root.NewPath(path)` or `op.NewPath(root, rel)` (tests/deserialization).
   There is no way to construct a Path with inconsistent `{root, rel, abs}`.

4. **RecoverySite is opaque.** Providers pass Paths in and get string IDs
   back. The recovery directory structure is an internal concern. Providers
   never construct recovery Paths or manipulate recovery directory entries.

5. **Zero data movement.** RecoverySite uses `Root.Rename` for file
   archival and restoration. The recovery directory is inside the authority
   boundary — same partition, same filesystem, O(1) rename.
