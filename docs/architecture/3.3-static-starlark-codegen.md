# Static Starlark Value Codegen

**Status:** draft

## Core Idea

Every Go type exposed to Starlark gets a `starlark.Value` implementation. A single wrapper type
(`bind.StaticValue`) provides the `starlark.Value` boilerplate. Type-specific parts — attribute
dispatch, marshaling, and unmarshaling — are provided by the wrapped type.

The point of view: **you give me a Go value, I give you a `starlark.Value` back.** Receiver
factories are replaced by value factories. All marshaling and unmarshaling is consolidated into
single locations with well-defined APIs.

```
        Go value
            │
    MarshalStarvalue()     ← per type (reflection or generated)
            │
            ▼
    ┌───────────────┐
    │  StaticValue  │      ← one wrapper for all types
    │               │
    │  Attr(name)   │──→  StarAttr(name)    ← per type dispatch
    │  AttrNames()  │──→  StarAttrNames()   ← per type list
    │  String()     │──→  fmt.Stringer      ← existing method
    │  Freeze()     │     (no-op)
    │  Truth()      │     (always True)
    │  Hash()       │     (unhashable)
    └───────────────┘
```

## Phasing

The work is split into two parts that share the same API surface.

### Part 1: Consolidate Conversion Logic (reflection)

Refactor the existing reflection-based code so that all marshaling and unmarshaling flows through
centralized functions with clean APIs:

- **`bind.Marshal()`** — Go → Starlark entry point (already exists, minor changes)
- **`bind.Unmarshal()` / `UnmarshalAs[T]()`** — Starlark → Go, single switch
- **`bind.CoerceAs[T]()`** — Go → Go slot coercion, single function
- **`bind.Marshaler` / `bind.Unmarshaler`** — interfaces for structured types
- **`bind.StarAttrs`** — interface for attribute dispatch
- **`bind.StaticValue`** — single wrapper implementing `starlark.Value`

The scattered conversion logic (`unmarshalValue`, `coerceSlotValue`, `marshalReflect`) is
consolidated into these locations. Reflection still powers the internals, but every conversion
enters through one of these entry points. The param registries, `StructValue`, and bridge
builders continue to work — they just route through the new APIs.

This part can be done incrementally within the current `extract-starlark-from-op` branch.

### Part 2: Static Codegen (eliminate reflection for module types)

Generate `StarAttrs`, `Marshaler`, and `Unmarshaler` implementations for all module-owned types.
The centralized functions from Part 1 check for these interfaces first and fall through to
reflection only for third-party types.

- Generated `StarAttr` switches replace `StructValue` reflection field walk
- Generated `bridge<Method>` functions replace `buildMethodBridge` reflection
- Generated `Do` methods replace `buildDoFunc` + `coerceArgs` reflection
- Parameter names baked into generated code; param registries dissolve
- `StructValue`, `discoverMethods`, `typeCache` no longer needed for module types

The API surface established in Part 1 does not change. Only the implementations behind the
interfaces shift from reflection to generated code.

## Motivation

The current `StructValue` + reflection path has three costs:

1. **Parameter name loss.** Go reflection does not preserve parameter names. The current system
   works around this with a side registry (`typeParamsRegistry`, `receiverParamsRegistry`) populated
   by generated `MethodParams` maps. The parameter names are already extracted from the AST at
   generation time — but then stored as data for reflection to consume later.

2. **Runtime cost.** Every `Attr()` call on a `StructValue` goes through `reflect.Value.Field()` or
   `reflect.Value.MethodByName()`, plus generic marshaling via `marshalReflect`. Method bridges
   are built with `reflect.Method.Call()`.

3. **Indirection.** The `ReceiverFactory` interface carries method metadata (`MethodParams`,
   `MethodParamsFor`) solely to feed the reflection bridge. The factory, the param registries, and
   the bridge builder are all intermediaries between the Go type and Starlark.

Part 1 addresses cost 3 by consolidating the entry points. Part 2 addresses costs 1 and 2 by
eliminating reflection for module-owned types entirely.

## Design

### Conversion Strategy

There are three tiers of type conversion, ordered by preference:

1. **Starlark converters** — for primitive types. Starlark provides built-in converters in both
   directions. These are always preferred when the Go type maps directly to a Starlark primitive.

2. **`bind.Marshaler` / `bind.Unmarshaler`** — for module-owned structured types that need
   attribute dispatch (fields, methods). Generated code implements these interfaces.

3. **`bind.Marshal()` reflection fallback** — for third-party types outside the module boundary.

The generator picks the tier based on the Go type it encounters in the AST:

| Go type | Marshal (Go → Starlark) | Unmarshal (Starlark → Go) |
|---------|------------------------|--------------------------|
| `string` | `starlark.String(v)` | `bind.UnmarshalAs[string](v)` |
| `bool` | `starlark.Bool(v)` | `bind.UnmarshalAs[bool](v)` |
| `int64` | `starlark.MakeInt64(v)` | `bind.UnmarshalAs[int64](v)` |
| `uint64` | `starlark.MakeUint64(v)` | `bind.UnmarshalAs[uint64](v)` |
| `float64` | `starlark.Float(v)` | `bind.UnmarshalAs[float64](v)` |
| `[]byte` | `starlark.Bytes(v)` | `bind.UnmarshalAs[[]byte](v)` |
| Module struct | `v.MarshalStarvalue()` | `bind.UnmarshalAs[*T](v)` |
| Third-party struct | `bind.Marshal(v)` | reflection fallback |

Marshaling (Go → Starlark) uses direct Starlark constructors for primitives and
`MarshalStarvalue()` for structured types — the generator picks the right call per field.

Unmarshaling (Starlark → Go) always goes through `bind.Unmarshal()` — a single switch that
converts any `starlark.Value` to its natural Go representation. `bind.UnmarshalAs[T]()` adds a
type assertion on top. Generated bridges only specify the target type; the conversion logic lives
in one place.

For module structs, `Unmarshal()` returns the `StarAttrs` inner value from the `StaticValue`
wrapper. `UnmarshalAs[*file.Resource](v)` then asserts the concrete type. No per-type unmarshal
switch is generated in the bridges.

`Marshaler` / `Unmarshaler` are reserved for types that are *more than a primitive* — they need
attribute access. If a type maps cleanly to a Starlark primitive, it should not implement these
interfaces.

### Shared Infrastructure (pkg/op/bind)

#### StarAttrs Interface

The contract that generated types implement:

```go
type StarAttrs interface {
    StarType() string
    StarAttr(name string) (starlark.Value, error)
    StarAttrNames() []string
}
```

#### StaticValue Wrapper

One wrapper for all generated types:

```go
type StaticValue struct {
    inner StarAttrs
}

func NewStaticValue(inner StarAttrs) *StaticValue

func (v *StaticValue) Type() string                              { return v.inner.StarType() }
func (v *StaticValue) Attr(name string) (starlark.Value, error)  { return v.inner.StarAttr(name) }
func (v *StaticValue) AttrNames() []string                       { return v.inner.StarAttrNames() }
func (v *StaticValue) Freeze()                                   {}
func (v *StaticValue) Truth() starlark.Bool                      { return starlark.True }
func (v *StaticValue) Hash() (uint32, error)                     { /* unhashable */ }
func (v *StaticValue) String() string                            { /* delegate to fmt.Stringer or build from attrs */ }

func (v *StaticValue) Inner() StarAttrs                          { return v.inner }
```

#### Marshaler and Unmarshaler Interfaces

Already defined in bind. Generated code satisfies them for structured types only:

```go
// Go → Starlark (for types that need attribute dispatch)
type Marshaler interface {
    MarshalStarvalue() (starlark.Value, error)
}

// Starlark → Go (for types that need structured unwrapping)
type Unmarshaler interface {
    UnmarshalStarvalue(starlark.Value) error
}
```

`Marshal()` checks in order: Starlark converter (primitives) → `Marshaler` interface (structured
types) → reflection fallback (third-party types).

#### Centralized Unmarshal

A single function converts any `starlark.Value` to its natural Go representation. This eliminates
per-bridge conversion logic — the primitive type switch lives in one place, not scattered across
generated code.

```go
// Unmarshal converts a starlark.Value to its natural Go type.
// Primitives become Go primitives. StaticValue unwraps to the inner Go value.
func Unmarshal(v starlark.Value) (any, error) {
    switch v := v.(type) {
    case starlark.String:
        return string(v), nil
    case starlark.Bool:
        return bool(v), nil
    case starlark.Int:
        if i, ok := v.Int64(); ok { return i, nil }
        if u, ok := v.Uint64(); ok { return u, nil }
        return nil, fmt.Errorf("int overflow")
    case starlark.Float:
        return float64(v), nil
    case starlark.Bytes:
        return []byte(v), nil
    case *StaticValue:
        return v.Inner(), nil
    case starlark.NoneType:
        return nil, nil
    default:
        return nil, fmt.Errorf("cannot unmarshal %s", v.Type())
    }
}
```

A generic helper adds the type assertion:

```go
// UnmarshalAs converts a starlark.Value to the specified Go type.
func UnmarshalAs[T any](v starlark.Value) (T, error) {
    raw, err := Unmarshal(v)
    if err != nil {
        var zero T
        return zero, err
    }
    t, ok := raw.(T)
    if !ok {
        var zero T
        return zero, fmt.Errorf("expected %T, got %T", zero, raw)
    }
    return t, nil
}
```

Generated bridges use `UnmarshalAs[T]` with the target type known at generation time. The bridge
does not contain any type conversion logic — only the target type name.

### Generated Code Per Type

Each module-owned type gets three groups of generated methods.

#### 1. Marshaler — Go to Starlark

```go
// file/resource.gen.go

func (r *Resource) MarshalStarvalue() (starlark.Value, error) {
    return bind.NewStaticValue(r), nil
}
```

#### 2. StarAttrs — Attribute Dispatch

Fields become direct access. Zero-arg methods become direct calls. Parameterized methods become
builtins with baked-in parameter names.

```go
var resourceAttrNames = []string{
    "checksum", "device", "exists", "fragment", "host", "id",
    "inode", "mod_time", "mode", "opaque", "origin_id", "path",
    "scheme", "size", "source_path", "uri",
}

func (r *Resource) StarType() string        { return "file_resource" }
func (r *Resource) StarAttrNames() []string { return resourceAttrNames }

func (r *Resource) StarAttr(name string) (starlark.Value, error) {
    switch name {
    // Fields — direct access, no reflection
    case "source_path":
        return r.SourcePath.MarshalStarvalue()  // op.Path — also generated
    case "inode":
        return starlark.MakeUint64(r.Inode), nil
    case "device":
        return starlark.MakeUint64(r.Device), nil
    case "size":
        return starlark.MakeInt64(r.Size), nil
    case "mode":
        return starlark.MakeUint(uint(r.Mode)), nil
    case "mod_time":
        return bind.Marshal(r.ModTime)  // time.Time — third party, reflection fallback
    case "checksum":
        return starlark.String(r.Checksum), nil
    // Zero-arg methods — direct calls
    case "uri":
        return starlark.String(r.URI()), nil
    case "id":
        return starlark.String(r.ID()), nil
    case "origin_id":
        return starlark.String(r.OriginID()), nil
    case "scheme":
        return starlark.String(r.Scheme()), nil
    case "opaque":
        return starlark.String(r.Opaque()), nil
    case "host":
        return starlark.String(r.Host()), nil
    case "path":
        return starlark.String(r.Path()), nil
    case "fragment":
        return starlark.String(r.Fragment()), nil
    case "exists":
        return starlark.Bool(r.Exists()), nil
    default:
        return nil, bind.NoSuchAttrError("file_resource", name)
    }
}
```

#### 3. Unmarshaler — Starlark to Go

Two paths: unwrap a `StaticValue` holding the same type, or construct from a primitive.

```go
func (r *Resource) UnmarshalStarvalue(v starlark.Value) error {
    // Fast path: unwrap StaticValue holding same type
    if sv, ok := v.(*bind.StaticValue); ok {
        if src, ok := sv.Inner().(*Resource); ok {
            *r = *src
            return nil
        }
    }
    // Construction path: string → ResourceFromValue
    if s, ok := v.(starlark.String); ok {
        res, err := ResourceFromValue(string(s))
        if err != nil {
            return err
        }
        *r = res
        return nil
    }
    return fmt.Errorf("cannot unmarshal %s as file_resource", v.Type())
}
```

### Generated Method Bridges — Replacing Receiver Factories

For provider types, `StarAttr` returns `starlark.Builtin` values for action methods. Parameter
names, type conversions, and Go calls are all baked in:

```go
// file/provider.gen.go

func (p *Provider) StarAttr(name string) (starlark.Value, error) {
    switch name {
    case "write":
        return starlark.NewBuiltin("file.write", p.bridgeWrite), nil
    case "read":
        return starlark.NewBuiltin("file.read", p.bridgeRead), nil
    // ...
    }
}

func (p *Provider) bridgeWrite(
    thread *starlark.Thread, _ *starlark.Builtin,
    args starlark.Tuple, kwargs []starlark.Tuple,
) (starlark.Value, error) {

    var pathVal, contentVal starlark.Value
    if err := starlark.UnpackArgs("write", args, kwargs,
        "path", &pathVal, "content", &contentVal); err != nil {
        return nil, err
    }

    // UnmarshalAs — target type known from AST, conversion logic in bind
    path, err := bind.UnmarshalAs[string](pathVal)
    if err != nil { return nil, fmt.Errorf("file.write: path: %w", err) }
    content, err := bind.UnmarshalAs[string](contentVal)
    if err != nil { return nil, fmt.Errorf("file.write: content: %w", err) }

    // Direct Go call — no reflect.Value.MethodByName
    result, tombstone, err := p.Write(path, content)
    if err != nil {
        return nil, err
    }

    _ = tombstone // compensation handling still needed
    return result.MarshalStarvalue()
}
```

### Slot Filling and Graph Execution

There are two paths where values are converted into Go method parameters. Both are currently
reflection-heavy. Static codegen replaces the reflection in both with a shared centralized
function and generated type-specific dispatch.

#### The Two Paths

**Starlark immediate mode** — a Starlark script calls a provider method directly:

```
starlark.UnpackArgs() → []starlark.Value
  → unmarshalValue(sv, reflect.Value)     ← 200-line reflection switch
  → method.Func.Call([]reflect.Value)     ← reflection call
  → classifyReturn() → marshalReflect()   ← reflection marshal
```

**Graph execution** — the executor resolves slots from upstream results and calls `Method.Do()`:

```
node.ResolvedSlots() → map[string]any
  → coerceArgs()
    → coerceSlotValue(any, reflect.Type)  ← reflection: AssignableTo? ConvertibleTo? Map→Struct?
  → callMethod() → reflect.Method.Call()  ← reflection call
  → classify*Return()                     ← reflection inspect
```

Both paths solve the same problem — convert a value of unknown type into a typed Go method
parameter — using a different input type (`starlark.Value` vs `any`).

#### Centralized Conversion Functions

Two shared functions replace the reflection switches. Each handles its input type with a single
switch in one place:

```go
// Starlark path: starlark.Value → natural Go type
func Unmarshal(v starlark.Value) (any, error) { /* see above */ }
func UnmarshalAs[T any](v starlark.Value) (T, error) { /* type assertion wrapper */ }

// Graph path: any (Go value from upstream) → typed parameter
func CoerceAs[T any](v any) (T, error) {
    // Fast path: already the right type
    if t, ok := v.(T); ok {
        return t, nil
    }
    // Convertible (int → FileMode, etc.)
    // Map → struct
    // Slice coercion
    // Constructor registry
    var zero T
    return zero, fmt.Errorf("cannot coerce %T to %T", v, zero)
}
```

#### Generated Bridges Replace Both Paths

For Starlark immediate mode, the generated bridge uses `UnmarshalAs[T]`:

```go
func (p *Provider) bridgeWrite(...) (starlark.Value, error) {
    starlark.UnpackArgs(...)
    path, err := bind.UnmarshalAs[string](pathVal)
    content, err := bind.UnmarshalAs[string](contentVal)
    result, _, err := p.Write(path, content)
    return result.MarshalStarvalue()
}
```

For graph execution, the generated `Do` method uses `CoerceAs[T]`:

```go
func (m *fileWriteMethod) Do(ctx *op.Context, slots map[string]any) (op.Result, op.Complement, error) {
    provider := m.factory.GetOrCreateProvider(*ctx).(*Provider)

    path, err := bind.CoerceAs[string](slots["path"])
    if err != nil { return nil, nil, fmt.Errorf("file.write: path: %w", err) }
    content, err := bind.CoerceAs[string](slots["content"])
    if err != nil { return nil, nil, fmt.Errorf("file.write: content: %w", err) }

    result, tombstone, err := provider.Write(path, content)
    if err != nil { return nil, nil, err }

    _ = tombstone // compensation
    return shadowResult(result, ctx.Catalog, ctx.NodeID), nil, nil
}
```

Both paths follow the same shape — the only difference is the input type:

| | Starlark path | Graph path |
|--|--------------|------------|
| Input | `starlark.Value` | `any` (Go value from upstream) |
| Convert | `bind.UnmarshalAs[T]()` | `bind.CoerceAs[T]()` |
| Call | direct | direct |
| Return | `result.MarshalStarvalue()` | `return result` (already Go) |

#### What Doesn't Change

- **`node.ResolvedSlots()`** — map plumbing that resolves promises and proxy slots; no types
- **`FillSlotsFromData()`** — fills missing slots from context data; no types
- **`initCallableSlots()`** — callable adaptation (Starlark func → Go func) is a separate concern

#### What This Replaces

| Current | Replaced by |
|---------|------------|
| `unmarshalValue()` — 200-line reflection switch | `bind.Unmarshal()` — single shared switch |
| `coerceSlotValue()` — reflection coercion hierarchy | `bind.CoerceAs[T]()` — single shared function |
| `coerceArgs()` — reflection parameter loop | Generated `Do` with `CoerceAs[T]` calls |
| `buildDoFunc()` — closure over reflection dispatch | Generated `Do` method per action |
| `callMethod()` — `reflect.Method.Call()` | Direct Go method call |
| `classifyFallibleReturn()` / `classifyCompensableReturn()` | Generated code that knows return types |

## Type Processing and Deduplication

The code generator processes all types in a single pass to avoid duplicate generation.

### Algorithm

1. **Seed** the work queue with all root types (resources and providers across all packages).
2. **BFS**: for each type, scan exported fields and method signatures. For each struct type
   encountered that is within the module boundary, enqueue it if not already seen.
3. **Collect** results into `map[package_path][]type`.
4. **Emit** one generated file per package containing all implementations for types in that package.

### Deduplication Key

Fully qualified type name (e.g., `pkg/op.Path`). First encounter registers it; subsequent
encounters are no-ops.

### Boundary Rules

| Encountered type | Action |
|-----------------|--------|
| Module-owned struct, not yet seen | Enqueue — will generate StarAttrs + Marshaler + Unmarshaler |
| Module-owned struct, already seen | Skip |
| Stdlib with known conversion | Emit direct starlark constructor (e.g., `starlark.MakeInt64`) |
| Third-party struct | Emit `bind.Marshal()` fallback — reflection handles it |
| Interface | `bind.Marshal()` fallback — concrete type unknown at generation time |

### Embedded Types

Embedded types like `ResourceBase` are not generated as standalone `StarAttrs` implementors
(they are never used as standalone Starlark values). Their promoted methods are inlined into the
embedding type's `StarAttr` switch.

### Output Layout

One generated file per package that owns types needing generation:

```
pkg/op/starvalue.gen.go                    → Path
pkg/op/provider/file/starvalue.gen.go      → Resource
pkg/op/provider/file/gen/provider.gen.go   → Provider bridges
pkg/op/provider/mem/starvalue.gen.go       → Resource
pkg/op/provider/git/starvalue.gen.go       → Resource
...
```

## What Changes Per Part

### Part 1: Consolidate (reflection preserved)

| Current | Part 1 change |
|---------|--------------|
| `unmarshalValue` — 200-line switch in `marshal.go` | Consolidated into `bind.Unmarshal()` / `UnmarshalAs[T]()` |
| `coerceSlotValue` — reflection coercion in `action.go` | Consolidated into `bind.CoerceAs[T]()` |
| `marshalReflect` — scattered marshal calls | Routed through `bind.Marshal()` checking `Marshaler` first |
| `StructValue` | Implements `StarAttrs`; wrapped by `StaticValue` |
| `buildMethodBridge` / `buildDoFunc` | Refactored to use `UnmarshalAs[T]` / `CoerceAs[T]` |
| Param registries | Unchanged (still feed reflection bridges) |

Part 1 establishes the API surface. Reflection still powers the internals.

### Part 2: Static Codegen (reflection eliminated for module types)

| After Part 1 | Part 2 change |
|-------------|--------------|
| `StructValue` implementing `StarAttrs` via reflection | Generated `StarAttr` switch per type — direct field access |
| `typeParamsRegistry` / `RegisterTypeParams` | Param names baked into generated bridges |
| `receiverParamsRegistry` | Not needed |
| `typeCache` (struct introspection) | Not needed |
| `discoverMethods()` reflection scan | Generated `StarAttrNames` + `StarAttr` cases |
| Reflection-based `buildMethodBridge` | Generated `bridge<Method>` functions |
| Reflection-based `buildDoFunc` + `coerceArgs` | Generated `Do` method per action |
| `classify*Return` reflection inspection | Generated code that knows return types |
| `WrapProviderInExecutingReceiver` | `provider.MarshalStarvalue()` |
| `ReceiverFactory.MethodParams()` / `MethodParamsFor()` | Not needed |
| `marshalReflect` for module types | Generated `MarshalStarvalue()` |

## What Remains After Both Parts

- **`bind.Marshal()`** — entry point; checks `Marshaler` first, reflection fallback for
  third-party types
- **`bind.Unmarshal()` / `UnmarshalAs[T]()`** — centralized Starlark → Go conversion (one switch)
- **`bind.CoerceAs[T]()`** — centralized Go → Go slot coercion (one function)
- **`marshalReflect`** — fallback path for `time.Time`, `os.FileMode`, and other external types
- **`StaticValue`** — the one shared wrapper
- **`node.ResolvedSlots()` / `FillSlotsFromData()`** — slot plumbing unchanged
- **`initCallableSlots()`** — callable adaptation unchanged
- **Compensation/tombstone handling** — orthogonal concern, still present in bridges
- **Provider creation** — something still instantiates the Go provider from a `Context`; the
  factory shrinks to just this role
