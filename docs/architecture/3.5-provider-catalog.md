## Purpose

Each provider owns a Starlark namespace (`file.*`, `archive.*`, `git.*`, …) and contributes a set of actions or a
resource scheme to the execution graph. This catalog is the **index of record** for which providers exist, the namespace
each owns, its access role zone, and — where one has been written — a link to its dedicated design document. New
providers are added here as a row; providers substantial enough to warrant a full design doc get a numbered `3.5.x`
sub-document.

## Access role zones

A provider's `+devlore:access` directive declares **when** its actions are reachable (see
[3.2-projected-provider-api.md](3.2-projected-provider-api.md) and the role-zone model):

- **planned** — plan-time graph construction; the action contributes nodes to the graph and runs during execution.
- **immediate** — executes at call time (no graph node).
- **both** — available in either context.
