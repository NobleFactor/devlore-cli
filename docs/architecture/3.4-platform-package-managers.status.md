# Status: Platform and Package Management

**Architecture document:** [3.4-platform-package-managers.md](3.4-platform-package-managers.md)

The design is settled (2026-06-04) and was **reversed to an op-free `pkg/platform`**: `platform.Platform` stays in
`pkg/platform` (importing nothing from `pkg/op`), `pkg/op` imports it, and there is **no `op.Platform` /
`platformData`** — identity serialization into the graph is deferred to
[#282](https://github.com/NobleFactor/devlore-cli/issues/282). The `pkg/platform` reshape (Composite router, purl
routing with construction-time normalization, the native driver catalog, per-leaf state-query verification, named
`*Spec` factories) is **implemented and style-compliant**; the consumer migration and `pkg/platform` test rewrites
remain. It supersedes the stale "Darwin Package Manager Idempotence" treatment formerly in
[§3 Action Namespaces](3-operation-namespaces.md), which was OS-specific and built on the removed `Host` interface.
