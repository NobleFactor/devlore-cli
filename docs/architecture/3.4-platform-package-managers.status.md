# Status: Platform and Package Management

**Architecture document:** [3.4-platform-package-managers.md](3.4-platform-package-managers.md)

The design is settled (2026-06-04 design session) and recorded; implementation has not started. The document
describes the target model: `op.Platform` as a serializable capability, `op.PackageManager` as a per-platform
Composite router, purl routing with construction-time normalization, the native + shared-language driver catalog,
and state-query verification (no screen-scraping). It supersedes the stale "Darwin Package Manager Idempotence"
treatment formerly in [§3 Action Namespaces](3-operation-namespaces.md), which was OS-specific and built on the
removed `Host` interface.
