# Stop expanding struct_param in codegen

## Context

The reflection bridge (`buildMethodBridge`) assumes a 1:1 mapping between param names in `MethodParams` and Go method parameters. The codegen's `+devlore:struct_param` annotation expands struct fields into individual param names, breaking this assumption.

Example: `Sources.Analyze(cfg staranalysis.AnalysisConfig)` has 1 Go param, but the generated params list has 4 entries (`hotspots?`, `cyclomatic_threshold?`, `cognitive_threshold?`, `with_index?`). The bridge maps param[0] (`hotspots`) to `methodType.In(1)` (`AnalysisConfig`), then tries to unmarshal a bool into a struct.

**Fix**: Stop expanding struct fields. Emit Go param names. The marshaler already handles dict → struct.

## Data structure

`MethodParams map[string][]string` — unchanged. Just strings. The reflection bridge knows the Go types from the method signature. `unmarshalValue` handles `starlark.Dict` → Go struct.

Before:
```go
"Analyze": {"files", "hotspots?", "cyclomatic_threshold?", "cognitive_threshold?", "with_index?"},
```

After:
```go
"Analyze": {"files", "cfg?"},
```

## Affected methods (3 total)

All have `+devlore:struct_param cfg=AnalysisConfig`:

| Provider | Method | Go signature | Params before | Params after |
|----------|--------|-------------|---------------|--------------|

## Phase 1: Fix codegen — `generate.star` (noblefactor-ops)

**File**: `star/extensions/com.noblefactor.devlore.Actions/commands/generate.star`

In `build_method_descriptors` (lines 440-471): when a param name appears in `method_struct_params`, currently expands struct fields into individual param entries. Change to emit a single param entry with the original Go param name, marked optional:

```python
if p.name in method_struct_params:
    params.append({
        "name": p.name,
        "type": method_struct_params[p.name],
        "variadic": False,
        "doc": "",
        "optional": True,
        "default": "",
    })
```

This replaces the field expansion loop entirely.

## Phase 2: Add map → struct coercion — `action_reflect.go` (devlore-cli)

**File**: `pkg/op/action_reflect.go`

The planned → action path stores slot values as Go-native types via `unmarshalToAny`. A `starlark.Dict` becomes `map[string]any` in the slot. When `coerceSlotValue` tries to push `map[string]any` into `AnalysisConfig`, it needs a new case.

In `coerceSlotValue`, add between the "convertible" and "constructor" cases:

```go
// map[string]any → struct: iterate struct fields, coerce each value from map
if slotType.Kind() == reflect.Map && slotType.Key().Kind() == reflect.String &&
    targetType.Kind() == reflect.Struct {
    return coerceMapToStruct(slotValue, targetType)
}
```

`coerceMapToStruct` iterates the target struct's exported fields, looks up the snake_case key in the map, and recursively coerces each value.
